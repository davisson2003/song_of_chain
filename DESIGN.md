# Song of BNBCHAIN — 设计说明

本文档说明 **Song of BNBCHAIN**（原 BSC Chain Music 概念）在架构、链上数据、声音映射、交互与可视化等方面的设计决策，便于维护与二次开发。

---

## 1. 项目定位

- **一句话**：把 BNB Smart Chain（BSC）的**区块与交易活动**实时映射为**多轨电子乐**，并支持历史区间回放。
- **形态**：单文件 `index.html`，通过 CDN 加载 React、Babel、Tone.js，无需构建步骤即可本地或静态托管打开。
- **品牌**：页眉标题为 **SONG OF BNBCHAIN**；中文界面下文档标题为 **BNBCHAIN 之歌**。

---

## 2. 技术架构

| 层级 | 技术 | 说明 |
|------|------|------|
| 表现层 | React 18（UMD）+ 内联 JSX | `babel-standalone` 在浏览器中转译 JSX |
| 音频 | Tone.js 14.x | 合成器、效果器、`Tone.Transport` 用于回放调度 |
| 数据 | JSON-RPC + WebSocket | `eth_getBlockByNumber`、`eth_subscribe`（`newHeads`） |
| 存储 | `localStorage` | 语言偏好 `song-of-chain-lang` |

**设计取舍**：不引入打包工具，降低上手成本；代价是单文件体积较大、逻辑集中在一处。

---

## 3. 链上数据层

### 3.1 区块获取

- **实时**：优先 **WebSocket** 订阅新区块头，再用 HTTP 拉取**完整区块**（含 `transactions`）。
- **兜底**：WS 失败或关闭时改为 **HTTP 轮询** `latest`，间隔约等于预期出块时间（`BLOCK_MS ≈ 450ms`）。
- **历史回放**：按区块号 **批量 JSON-RPC**（每批最多 20 个块），多批并行；失败时回退为逐块请求。

### 3.2 节点与限流

- **公共池**：多个 HTTP 端点轮询尝试；单次请求带超时，全部失败则抛出可本地化的错误。
- **NodeReal**：用户可选填 API Key，统一走 NodeReal 的 HTTP / WS，适合需要稳定配额的场景。

### 3.3 区块内用到的字段

- `transactions[]`：`from`、`to`、`input`（用于分类）
- `gasUsed` / `gasLimit`：计算 **Gas 利用率**（环境层与整体动态）
- `number`：区块高度（节拍感、BPM 估算、UI）

---

## 4. 交易分类

对区块内每笔交易计数三类（与以太坊风格一致）：

| 类型 | 判定 |
|------|------|
| **Deploy（部署）** | `to` 为空（合约创建） |
| **DeFi / 合约调用** | `to` 存在且 `input` 非 `0x` |
| **Transfer（转账）** | `to` 存在且 `input` 为 `0x`（简单价值转移） |

汇总得到 `transfers`、`defi`、`deploys`，以及 **总交易数 `total`**。该分类用于驱动不同音色轨道，**不区分具体代币**，以保持计算简单、延迟低。

---

## 5. 声音设计

### 5.1 多轨含义

| 轨道 | 声源类型 | 与数据的映射（概要） |
|------|----------|----------------------|
| Transfer | 三角波 PolySynth | 转账笔数 → 旋律音高与节奏细分 |
| DeFi | 锯齿波 PolySynth | 与 Transfer 和声关系（三度堆叠） |
| Deploy | MetalSynth | 部署次数 → 金属感打击 |
| Ambient | 正弦 Pad | Gas 利用率 → 长音与响度 |
| TX Beat | MembraneSynth | 总交易数 → 底鼓力度与音高衰减 |
| Bass（内部） | 低频正弦 | 按区块号强拍 / 次强拍点缀 |

### 5.2 音阶

预置多种 **调式音阶**（五声、蓝调、大调、全音等），仅影响 **音级集合**；用户可在界面切换，切换时会 **重置声部引导状态**，避免索引跨调式错乱。

### 5.3 声部引导（Voice-leading）

- 为避免“数字跳变导致旋律乱飞”，对 Transfer / Deploy 等轨使用 **按块限制步长**（每次在音阶索引上最多移动 2 步）。
- DeFi 轨相对 Transfer 固定 **三度音程** 关系，保持协和感。

### 5.4 动态与平滑

- **实时**：按块更新；可选 **每 N 块** 才触发一次音符（节流），减轻 BSC 高频出块带来的密度。
- **历史回放**：对力度做 **滑动窗口平均**（`smoothNotes`），听感更连贯。

### 5.5 效果链

- 主旋律/和声等走 **延迟 + 混响**；底鼓可走 **干声** 以保留冲击力（具体以代码中连线为准）。

## 6. 运行模式

| 模式 | 行为 |
|------|------|
| **实时（Live）** | 订阅或轮询新区块，逐块处理并播放；可选将本段会话缓冲为 **会话回放**。 |
| **历史（Replay）** | 用户输入起止区块 → 三阶段：**拉块 → 转音符 + 平滑 → Transport 定时播放**；支持倍速、循环、暂停/继续。 |
| **会话回放** | 停止实时播放后，对本次 Live 录下的音符序列做 Transport 回放（同样可调速、循环）。 |

**设计要点**：实时路径强调低延迟；回放路径强调 **时间轴稳定**（预加载后由 `Tone.Transport` 调度，减轻抖动）。

---

## 7. 地址独奏（Address Solo）

**目的**：只听**某一钱包地址**在链上的“戏份”，而非整链统计。

- **过滤规则**：只统计 `from` 或 `to` 等于该地址（小写规范化）的交易，再套用 **同一套** Deploy / DeFi / Transfer 分类。
- **空块**：该块内若没有任何相关交易，则进入 **soloEmpty** 状态：主旋律、和声、部署、底鼓、贝斯 **静音**，仅保留 **极弱 Ambient**（与全链模式相比压低 Gas→音量映射），避免完全死寂又保留区块上下文感。
- **校验**：地址须为 `0x` + 40 位十六进制；非法则禁止开始播放并提示错误。

---

## 8. 粒子可视化（伪 3D）

**目的**：用轻量 **Canvas 2D** 表现“区块在流动”，避免引入完整 Three.js 构建链。

- **触发**：每当当前展示区块 **`blockN` 变化**，按 **总交易量** 迸发粒子数量上限。
- **视觉参数**：**Gas 利用率** 影响色相与上升速度；粒子带 **深度权重**，屏幕坐标做透视混合，形成 **伪 3D**。
- **交互**：鼠标/触摸位置对粒子有微弱吸引（画布 `pointer-events: none`，不挡 UI）。
- **开关**：用户可关闭粒子层以省电或减弱干扰。

---

## 9. 国际化（i18n）

- **语言**：中文 / 英文，切换写入 `localStorage`。
- **范围**：标题、按钮、说明、错误信息、图例等；**音阶名、轨道标签** 随语言切换。
- **文档标题**：`<title>` 与首屏脚本根据语言设置，减少闪烁。

---

## 10. 移动端与 Web Audio

移动浏览器普遍要求 **在用户手势的同步调用链** 内启动或恢复 `AudioContext`。

**设计**：

- 在 **播放**、**会话回放**、**继续（暂停后）** 等关键点击中调用统一的 **`unlockAudioFromGesture()`**（内部 `Tone.start()` + 必要时 `context.resume()`）。
- 从 **后台回到前台** 时，若仍在播放且上下文被挂起，尝试 `resume()`。

**说明**：iOS 侧键静音时系统可能仍静音 Web 音频，属系统策略，应用层无法绕过。

---

## 11. UI / 信息架构

- **深色主题**：背景渐变、高对比度文字、等宽字体强调“链上/终端”气质。
- **层次**：粒子层 → 主内容（`z-index`）；控制区分 **RPC 配置 / 独奏与粒子 / 模式 / 回放区 / 钢琴卷帘 / 统计 / 混音 / 图例**。
- **状态展示**：连接状态（WS / 轮询 / 回放阶段）、BPM（由出块间隔滑动估计）、回放进度条。

---

## 12. 扩展与维护建议

- **多链**：抽象 RPC 端点与链 ID，复用同一套映射逻辑即可试听不同链的“性格”。
- **更细粒度数据**：若需内部交易、日志事件，需额外索引服务或节点能力，会偏离当前“纯 RPC 区块”的轻量假设。
- **真 3D**：若引入 Three.js，建议单独拆包与懒加载，避免拖慢首屏。
